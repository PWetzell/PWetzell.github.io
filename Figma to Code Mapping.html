Figma to Code Mapping

Every Figma component variant mapped to its CSS class and HTML element. The bridge between design and engineering.

Use this as the source of truth when translating designs to code. Every Figma variant maps to a CSS class and HTML element.

Naming Conventions

  • Figma uses PascalCase for property names: Type=Primary, State=Hover, Size=MD
  • CSS classes use kebab-case: .btn-primary, .btn-md
  • State modifiers use BEM-style: .is-active, .is-error, :hover, :focus
  • Semantic HTML first — use the right element before adding ARIA
⚠
Hover/focus states are never separate HTML elements — always CSS pseudo-classes. The Figma variants exist for visual documentation only.
